3-队列

基本概念

定义

一种操作首先的线性表，只允许在表的一端进行插入，而在表的另一端进行删除

基本操作

InitQueue(*Q) 初始化队列：构造一个空队列
DestoryQueue(*Q) 销毁队列：销毁并释放队列 Q 所占用的内存空间
EnQueue(*Q, x) 入队：若队列 Q 未满，x 加入成新的队尾
DeQueue(*Q, *x) 出队：若队列 Q 未空，删除当前队头，x 返回
GetHead(*Q, *x) 读队头：x 返回队头元素
QueueEmpty(*Q) 判断队列：判断 Q 是否为空，返回 bool 值

循环队列

  • 定义：将顺序队列臆造成为一个环状空间，把存储队列元素的表从逻辑上堪称一个环。当队首指针 Q.front = MaxSize - 1 后，再前进一个位置就自动到 0
  • 特点：克服了“假上溢”现象
  • 上溢：队列空间已满，而继续往后队列中插入元素，使数组越界而导致程序代码被破坏
  • 假上溢：由于队尾指针不断前移，超出向量空间。这时整个向量空间及队列使空的却产生“上溢”现象
  • 入队：rear = (rear + 1) MOD MaxSize
  • 出队：front = (front + 1) MOD MaxSize

区分队空、队满的方法

  • case1：牺牲一个单元（上述代码中的方法）
    • 队空：Q.rear == Q.front
    • 队满：(Q.rear + 1) % MaxSize == Q.front
    • 队列个数：N = (Q.rear + MaxSize - Q.front) % MaxSize
  • *case2：引入 size
    • 在 SqQueue 的结构体中引入“size”字段，表明队列的数据成员数量
    • 队空：Q.size == 0
    • 队满：Q.size == MaxSize、Q.rear == Q.front
  • *case3：引入 tag
    • 在 SqQueue 的结构体中引入“tag”字段，删除操作令 tag 为 0，增加令 tag 为 1
    • 队空：front == rear && tag == 0
    • 队满：front == rear && tag == 1

双端队列

  • 定义：允许从两端插入、两端删除的线性表
  • *分类：
    • 正常的双端队列
    • 输入受限的双端队列
    • 输出受限的双端队列

队列的应用

  • 队列在层次遍历中的应用 → 使用队列使为了保存下一步的处理顺序
  • 队列在操作系统中的应用 → 数据缓冲区
  • *队列在排序算法的应用 → 基数排序

栈和队列的共同点

逻辑结构相同，都可以采用顺序存储或链式存储，均只匀速在端点处插入和删除元素

    1. 定义

      中缀表达式 前缀表达式 后缀表达式

    2. 中缀 → 后缀 手动算法（转换）

      1. 确定中缀表达式中各个运算符的运算顺序。（原则：左优先）
      2. 确定下一个运算符，按照【左操作数右操作数运算符】的方式足和成一个新的操作数
      3. 如果还有运算符没被处理，就继续执行 1 步骤


    目的： 没有括号，计算机易识别，速度快

    部分 步骤
    Part I
    机器算法思想    		 初始化一个栈，用于保存暂时还不能确定运算顺序的运算符。
    从左到右各个元素，直到末尾。可能遇到三种情况：
    case1：遇到操作数。直接加入后缀表达式。
    case2：遇到界限符。遇到"("直接入栈；遇到")"则依次弹出栈内运算符并加入后缀表达式，直到弹出"("为止。
    case3：遇到运算符。依次弹出栈中优先级高于或等于当前运算符的所有运算符，并加入后缀表达式，若碰到"("或栈空则停止。之后再把当前运算符入栈。
    按上述方法处理完所有字符后，将栈中剩余运算符一次弹出，并加入后缀表达式。
    Part II
    后缀算法思想    		 a - 从左往右扫描下一个元素，知道处理完所有元素
    b- 若扫面到操作数压入栈，并返回执行 a；否则执行 c
    c - 若扫描到运算符，则弹出两个栈顶元素，执行相应运算，运算结果压回栈顶，随后执行 a。（注意：先弹出的是左操作数）
    Part III
    实际算法想（结合）    		 a - 初始化两个栈，操作数栈和运算符栈
    b - 若扫描到操作数，压入操作数栈；若扫描到运算符或界限符，则按照"中缀转后缀"相同的逻辑压入运算符栈
    c - 期间也会弹出运算符，没当弹出一个运算符时，就需要再弹出两个操作数栈的栈顶元素并执行对应运算，运算结果再压回操作数栈
    1. 中缀 → 前缀
      1. 确定中缀表达式中哥哥运算符的运算顺序。（原则：右优先）
      2. 选择下一个运算符，按照【运算符左操作数右操作数】的方式足和成一个新的操作柱
      3. 如果还有运算符没被处理，就继续 b
        前缀算法：从右往左扫描下一个元素，知道处理完所有元素，其余步骤和后缀雷同（注意：先弹出的是左操作数，后弹出的是右操作数）

  2. 递归通过栈实现非递归

    • 使用情况：可以把原始问题转换为“属性相同”，但“规模较小”的问题
    • 递归和栈：递归调用时，“函数调用栈”可称为“递归工作栈”
    • 缺点：
      • 如果递归层数多，会导致栈溢出 → 递归复杂度越高，则空间复杂度越高
      • 可能包含很多重复的运算 → 效率低

  3. *括号匹配

    • 算法思想：从左到右一次扫描所有字符，“左括号”入栈，“右括号”弹出栈顶元素并检查是否匹配
    • 匹配失败： - 左括号单身 - 右括号单身 - 左右不匹配

注意：不能说采用非递归方式重写递归程序时必须使用栈。

循环链表

定义

循环单链表和单链表的区别在于，表中最后一个节点的指针不是 NULL，而改为指向头节点，从而整个链表形成一个环

特点

  1. 判空条件不是头节点的指针是否为空，而是它是否等于头指针

  2. 往最后一个节点后插/删除节点时，循环链表就能正常执行；普通链表会报错，需要特殊处理

*静态链表

定义

分配一整片连续的内存空间，各个节点集中安置

组成

数据元素 + 游标；游标充当指针作用，游标是 x，则该节点的后继的位置为 x

地址

首节点存放地址 + 空间 * 游标

特点

优：增、删、操作不用移动大量元素
缺：不能随机存取，只能从头结点以此往后插找；容量固定不变

应用

不支持指针的低级语言；数据元素数量固定不变的场景

1-绪论

数据结构的基本概念

  • 数据结构
    相互之间存在一种或多种特定关系的数据元素结合

  • *数据对象
    拥有相同性质的数据元素的几何，是数据的一个子集

  • *数据元素
    数据的基本单位，通常作为一个整体进行考虑和处理

  • *数据项
    是构成数据元素的不可分割的最小单位，一个数据元素有多个数据项

  • *数据类型

    • 原子类型：其值不能再分的数据类型
    • 结构类型：其值可以再分解为若干成分的数据类型
    • 抽象数据类型 ADT(Abstract Data Type)
      定义数据的逻辑结构、定义运算

  • 三要素

    • 逻辑结构：数据元素之间的逻辑关系
      线性、集合、树、图
    • 数据结构再计算机中的表示 顺序、链式、索引、散列/Hash
    • 数据运算
    • 关系：逻辑结构独立于存储结构，存储结构是逻辑结构于 PC 的映射

  • 算法

    • 定义：对特定问题求解步骤的一种描述
    • 5 个特性：有穷性、确定性、可行性、输入、输出
    • 特质/正确性、可读性、健壮性、高效率和低存储量需求

  • 时间复杂度 O(n)

    • 定义：算法中基本运算的执行次数量级（最坏情况）

    • 规则：

    • 排序：

      只挑循环中的一个基本操作分析它的执行次数于 n 的关系即可
      多层嵌套循环时，只考虑最深层的循环
      计算时，一般找到循环头 i 与 n 的关系 --> i ≤ f(n)

    • 分类：最坏时间复杂度、最好时间复杂度、平均时间复杂度

  • 空间复杂度 S(n)

    • 定义：算法所消耗的存储空间
    • 规则：同 O(n)
    • *术语：原地工作算法所需的辅助空间时常量
    • 原则：
      1. 出现存储空间大小和问题规模相关的变量
      2. 递归函数，取决于递归调用深度
