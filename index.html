  1. 栈在表达式求值中的应用

    1. 定义

      中缀表达式 前缀表达式 后缀表达式
      $a+b$ $+ a b$ $ab+$
      $a+b-c$ $-+ab$ $ab+c-$
      $a+b-c\times d$ $-+ab\times cd$ $ab+cd\times -$

    2. 中缀 → 后缀
      手动算法（转换）

      1. 确定中缀表达式中各个运算符的运算谁徐。（原则：左优先）
      2. 确定下一个运算符，按照【左操作数右操作数运算符】的方式足和成一个新的操作数
      3. 如果还有运算符没被处理，就继续执行 a 步骤


    目的： 没有括号，计算机易识别，速度快
    Part I
    机器算法思想    		 初始化一个栈，用于保存暂时还不能确定运算顺序的运算符。
    从左到右各个元素，直到末尾。可能遇到三种情况：
    case1：遇到操作数。直接加入后缀表达式。
    case2：遇到界限符。遇到”(“直接入栈；遇到”)”则一次弹出站内运算符并加入后缀表达式，知道弹出”(“为止。
    case3：遇到运算符。一次弹出栈中优先级高于或等于当前运算符的所有运算符，并加入后缀表达式，若碰到”(“或栈空则停止。之后再把当前运算符入栈。
    按上述方法处理完所有字符后，将栈中剩余运算符一次弹出，并加入后缀表达式。
    Part II
    后缀算法思想    		 a - 从左往右扫描下一个元素，知道处理完所有元素
    b- 若扫面到操作数压入栈，并返回执行 a；否则执行 c
    c - 若扫描到运算符，则弹出两个栈顶元素，执行相应运算，运算结果压回栈顶，随后执行 a。（注意：先弹出的是左操作数）
    Part III
    实际算法想（结合）    		 a - 初始化两个栈，操作数栈和运算符栈
    b - 若扫描到操作数，压入操作数栈；若扫描到运算符或界限符，则按照”中缀转后缀”相同的逻辑压入运算符栈
    c - 期间也会弹出运算符，没当弹出一个运算符时，就需要再弹出两个操作数栈的栈顶元素并执行对应运算，运算结果再压回操作数栈
    1. 中缀 → 前缀
      1. 确定中缀表达式中哥哥运算符的运算顺序。（原则：右优先）
      2. 选择下一个运算符，按照【运算符左操作数右操作数】的方式足和成一个新的操作柱
      3. 如果还有运算符没被处理，就继续 b
        前缀算法：从右往左扫描下一个元素，知道处理完所有元素，其余步骤和后缀雷同（注意：先弹出的是左操作数，后弹出的是右操作数）

  2. 递归通过栈实现非递归

    • 使用情况：可以把原始问题转换为“属性相同”，但“规模较小”的问题
    • 递归和栈：递归调用时，“函数调用栈”可称为“递归工作栈”
    • 缺点：
      • 如果递归层数多，会导致栈溢出 → 递归复杂度越高，则空间复杂度越高
      • 可能包含很多重复的运算 → 效率低

  3. *括号匹配

    • 算法思想：从左到右一次扫描所有字符，“左括号”入栈，“右括号”弹出栈顶元素并检查是否匹配
    • 匹配失败： - 左括号单身 - 右括号单身 - 左右不匹配

注意：不能说采用非递归方式重写递归程序时必须使用栈。

栈

基本概念、基本操作

定义

只允许在一端进行插入或删除操作的线性表

出栈顺序

有 n 个不同元素进栈，出栈元素不同排列的个数为：（卡兰特公式）

$$
\frac{1}{n+1}C_{2n}^n
$$

基本操作

InitStack(*S) 初始化栈：构造一个空栈 S，分配内存空间
DestroyStack(*S) 销毁栈：销毁并释放 S 所占的内存空间
Push(&S, x) 进栈：x 成为新栈项
Pop(&S, &x) 出栈：弹出栈顶并 x 返回
GetTop(&S, &x) 读栈：x 返回栈顶元素
StackEmpty(&S) 判断栈：判断 S 是否为空，返回 bool 值

  1. 共享栈

特点：函数运行结束后系统自动回收内存，所有操作时间复杂度为 O(1)

2-链表和顺序表的对比

线性表 线性表 顺序存储：随机存取、存储密度高；空间分配不方便，改变容量不方便
链表 线性表 链式存储：离散的小空间分配方便，改变容量方便；但是不可随机存取，存取密度低
基操 顺序表 链表
创 需要分配大片连续空间，若分配空间过小，则之后不方便拓展容量；
若分配空间过大，则浪费内存资源。		 只需分配一个头节点(甚至可以不要头节点只需要一个头指针)，扩展性更好
销 修改 length=0 一次删除各个节点(free)
增 插入元素后将后续元素移动，O(n) 只需修改指针，O(n)
删 删除元素要将后续元素移动，O(n) 只需子 u 该指针，O(n)
查 按位查：O(1)
按值查找：O(n)/O(log2n)[有序时]		 按位查找：O(n)
按值查找：O(n)

2-链表

单链表

定义

对每个链表节点，除了存放元素自身的信息之外，还需要存放一个指向其后继的指针

头节点：

  • 含头：

    1. 为了操作上的方便，在单链表第一个节点之前附加一个几点；
    2. 数据域可以不舍任何信息，但可以记录表长等信息；
    3. 声明时头节点只想的第一个节点为空，下一个节点才开始存放数据。

  • 不含：
    写代码麻烦。头节点只想的第一个节点就开始存放数据

循环链表

定义

循环单链表和单链表的区别在于，表中最后一个节点的指针不是 NULL，而改为指向头节点，从而整个链表形成一个环

特点

  1. 判空条件不是头节点的指针是否为空，而是它是否等于头指针
    •  bool isTail(LinkList L, LNode* p) {
   if (p->next == L)
     return true;
   else
     return false;
 }

*静态链表

定义

分配一整片连续的内存空间，各个节点集中安置

组成

数据元素 + 游标；游标充当指针作用，游标是 x，则该节点的后继的位置为 x

地址

首节点存放地址 + 空间 * 游标

特点

优：增、删、操作不用移动大量元素
缺：不能随机存取，只能从头结点以此往后插找；容量固定不变

应用

不支持指针的低级语言；数据元素数量固定不变的场景

2-顺序表

定义

用顺序存储的方式实现线性表

顺序存储

把逻辑上相邻的元素存储在物理位置上也相邻的存储单元中，元素之间的关系有存储单元的邻接关系来体现

地址特点

线性表 L 中的 an的地址是：

$$
LOC(C) + n \times sizeof(ElemType)
$$

式中：sizeof(ElemType) ——线性表中数据元素的大小

顺序表

  • 随机访问(在O(1)找到任意元素 i)
  • 存储密度高(如链式存储要存指针)

特点

  • 扩展容量不方便
  • 插入、删除不方便

2-基本概念

定义

相同数据类型的 n 个数据元素的有限序列。n 为表长，n=0 时为空表
写作：

$$
L = (a_1, a_2,…,a_i, a_{i+1}…,a_n)
$$

逻辑结构

序列，元素之间有先后关系

基本操作

  1. 创销

    • InitList(&L)：初始化表
    • DestroyList(&L)：销毁表

  2. 增删查

    • ListInsert(&L, i, e)：第 i 个位置插入元素 e（增）
    • ListDelete(&L, i, &e)：删除第 i 个位置的元素并用 e 表示（删）
    • LocateElem(L, e)：按给定关键字查找元素（查）
    • GetElem(L, i)：按为序 i 查找元素（查）

  3. 常用函数

    • Length(L)：返回表长
    • PrintList(L)：输出 L 所有元素
    • Empty(L)：判断 L 是否为空表

广义表

  • 定义：广义表时 n(n≥0)个元素 a1,a2,a3,…,an的有限序列

  • 特点：
    广义表中放松对表元素的源自限制，容许它们具有自身结构。
    广义表的第一个元素为广义表的表头，出去第一个元素后的其余元素组成冠以表的表尾。
    如 python 中：((1, 2, 3), [1, 2, 3], a, 1)，元组的各个元素可以有元组、序列、字符、数字

1-绪论

数据结构的基本概念

  • 数据结构
    相互之间存在一种或多种特定关系的数据元素结合

  • *数据对象
    拥有相同性质的数据元素的几何，是数据的一个子集

  • *数据元素
    数据的基本单位，通常作为一个整体进行考虑和处理

  • *数据项
    是构成数据元素的不可分割的最小单位，一个数据元素有多个数据项

  • *数据类型

    • 原子类型：其值不能再分的数据类型
    • 结构类型：其值可以再分解为若干成分的数据类型
    • 抽象数据类型 ADT(Abstract Data Type)
      定义数据的逻辑结构、定义运算

  • 三要素

    • 逻辑结构：数据元素之间的逻辑关系
      线性、集合、树、图
    • 数据结构再计算机中的表示
      顺序、链式、索引、散列/Hash
    • 数据运算
    • 关系：逻辑结构独立于存储结构，存储结构是逻辑结构于 PC 的映射

  • 算法

    • 定义：对特定问题求解步骤的一种描述
    • 5 个特性：有穷性、确定性、可行性、输入、输出
    • 特质/正确性、可读性、健壮性、高效率和低存储量需求

  • 时间复杂度 O(n)

    • 定义：算法中基本运算的执行次数量级（最坏情况）

    • 规则：

      $$
      T(n)=O(f(n))+O(g(n))=O(max(f(n),g(n)))
      $$

      $$
      T(n)=O(f(n))O(g(n))=O(f(n) \times g(n))
      $$

    • 排序：

      $$
      O(1)<(log_2n)<O(n)<O(nlog_2n)<O(n^2)<O(n^3)<O(2^n)<O(n!)<O(n^n)
      $$

      只挑循环中的一个基本操作分析它的执行次数于 n 的关系即可
      多层嵌套循环时，只考虑最深层的循环
      计算时，一般找到循环头 i 与 n 的关系 –> i ≤ f(n)

    • 分类：最坏时间复杂度、最好时间复杂度、平均时间复杂度

  • 空间复杂度 S(n)

    • 定义：算法所消耗的存储空间
    • 规则：同 O(n)
    • *术语：原地工作算法所需的辅助空间时常量
    • 原则：
      1. 出现存储空间大小和问题规模相关的变量
      2. 递归函数，取决于递归调用深度
